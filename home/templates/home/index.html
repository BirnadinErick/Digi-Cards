I always wanted to use flashcards for my study 🏫, but going out and buying some flashcards seems kinda "Nah-🙅‍♂️-Nah" to me, cuz they weren't of my taste 😋. How to?

Navigation🧭 is set in the hierarchy as Subjects👉Units👉Sub👉Units👉Flashcards. Digi-Cards uses MySQL for storage. Why🤔? Because using a SQL database will help relate flashcards according to our hierarchy.

You simply create a Subject; Then a Unit; Followed by a Sub Unit; Then at last the essence, a/many Flashcard(s).
You can manage each items via the backend. Just append /guardian to the URL. It will take you to the backend portal, where you can manage the entire database. Flashcards can be also modified via specially designated update page; Just append /update to the URL of the current flashcard.

Interested in the source code? Click the fork to 🍴 me in GitHub! Click the star to 🌟 me! Contribute! Or you can download the source code deploy your own one too 🎉!
